URL-Datei-Verbindung

Allgemein

Mit der URL-Datei-Verbindung sind Sie in der Lage, Passwort-Datenbanken zu öffnen, die sich auf einer entfernten Resource befinden und über eines der Internetprotokolle HTTP oder FTP erreichbar sind. Mit dieser Vorrichtung sind ein paar Einschränkungen verbunden. Die über HTTP geladenen Dateien können nur gelesen, nicht aber gespeichert werden. Über FTP geladene Dateien können nach Änderungen auch gespeichert werden, jedoch ist es gegenwärtig nicht möglich, neue Dateien anzulegen.

Sicherheitsbedenken

Generell ist es sehr gefährlich, sicherheitssensible Daten auf einer öffentlich zugänglichen Resource zu hinterlegen, wie gut die verwendete Verschlüsselungstechnik auch immer sein mag. Diese Vorrichtung wurde für Leute geschaffen, die sie unbedingt brauchen oder die möglicherweise ein privates Netz benutzen, das sie für sicher genug halten.

Falls Sie eine Passwort-Datenbank auf eine Internet-Resource legen wollen, sollten Sie unbedingt darauf achten, die bestmöglichen Sicherheitseinstellungen für die Datei zu verwenden. Das Zugangspasswort (geheimer Schlüssel) sollte nach meiner Überzeugung ein zufallsgenerierter Schlüssel von mindestens 10 Zeichen Länge sein. Worte aus dem Wörterbuch müssen unbedingt vermieden werden!

Kontrollelemente

Das editierbare Auswahlfeld URL akzeptiert jeden gültigen URL-Ausdruck. Das Feld erinnert darüber hinaus bis zu 8 erfolgreich durchgeführte Aufträge, wenn die Option "Liste der zuletzt geöffneten Dateien verwenden" in den Programm-Optionen aktiviert ist.

Die Löschen-Schaltfläche entfernt alle Einträge aus der URL-Auswahlliste.

Durchführung

Die OK-Schaltfläche startet die Durchführung und schließt diesen Dialog. Abhängig von Ihrer Internet-Verbindung kann es einige Sekunden dauern, bis eine Antwort angezeigt wird. (Sie können den Arbeitsstatus an dem kleinen Radzeichen in der rechten unteren Ecke des Programmfensters erkennen.)

Wenn Sie eine FTP-Adresse eingegeben haben, wird die folgende Antwort wahrscheinlich eine Login-Anfrage der FTP-Liegenschaft sein. Der Benutzername und das Passwort dazu werden in ein einziges Textfeld eingegeben, und zwar in der Schreibform <Benutzer>:<Passwort>. Beispiel: Wenn der Benutzername "lolamontez" und das Passwort "123456" ist, so lautet die korrekte Eingabe lolamontez:123456. Zu Ihrem Komfort erinnert JPasswords die FTP-Logins für die Dauer einer einzelnen Programmsitzung.

Nachdem eine Verbindung zu der Datei aufgebaut wurde, erfolgt die Passwort-Abfrage und die Datenbank kann in den Arbeitsbereich geladen werden.
